# 1. Purpose

This document defines the project scope for Version 1.0 of the T.N. Memorial School Digital Platform. It clearly identifies features included in the first release and those planned for future versions.

---

# 2. Project Scope

Version 1.0 includes the development of:

- Public School Website
- School ERP
- Authentication System
- Dashboard
- Student Management
- Teacher Management
- Attendance Management
- Examination Management
- Result Management
- Fee Management
- Notice Management
- Timetable Management
- Reports & Analytics
- User & Role Management
- Audit Logs
- System Settings

---

# 4. Out of Scope (Version 1.0)

The following features are intentionally excluded from Version 1.0:

- Mobile Applications (Android / iOS)
- Online Payment Gateway
- SMS Gateway
- WhatsApp Integration
- AI Chatbot
- Biometric Attendance
- RFID Attendance
- Hostel Management
- GPS Transport Tracking
- Video Conferencing
- Online Examination System

# 6. Scope Constraints

- Budget limitations
- Project timeline
- Available resources
- Infrastructure availability
- User training requirements

---

# 7. Scope Assumptions

- Management approval is available.
- Required hardware is available.
- Internet connectivity is stable.
- Users will participate in testing.

---

# 8. Scope Change Management

Any new feature request shall:

1. Be documented.
2. Be reviewed by the Project Manager.
3. Be evaluated by the Tech Lead.
4. Be approved before implementation.
5. Be added to the Product Backlog if deferred.

---

# 9. Acceptance Criteria

Version 1.0 will be accepted when:

- All in-scope modules are completed.
- Functional testing is passed.
- Security validation is completed.
- Documentation is approved.
- Deployment is successful.
